## Responsibilities

### 1. Schema Design

* Designed a complete SQLite relational database schema consisting of 17 tables
* Modeled all core entities including:
  `users`, `user_preferences`, `contents`, `authors`, `topics`, `topic_subtags`,
  `time_sessions`, `recommendations`, `bookmarks`, `progress`, `user_stats`,
  `achievements`, `user_achievements`, `content_topics`, `user_topics`,
  `onboarding_questions`, `user_onboarding_answers`
* Implemented many-to-many relationships using junction tables:

  * `content_topics` (contents to topics)
  * `user_topics` (users to topics, with preference weight)
  * `user_achievements` (users to achievements)
* Designed one-to-many relationships:

  * users to time_sessions, bookmarks, progress, and user_stats
  * authors to contents
  * topics to topic_subtags
  * time_sessions to recommendations
* Added performance indexes on frequently queried fields:
  `duration_minutes`, `difficulty`, `topic_id`, `session_id`

---

### 2. Constraints & Validation

* Applied data integrity constraints across all tables:

  * CHECK, UNIQUE, NOT NULL
* Key validation rules:

  * available_minutes: 1–240
  * duration_minutes: 1–720
  * avg_rating: 0–5
  * completion_percentage: 0–100
  * weight: 1–1
